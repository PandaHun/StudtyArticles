## 1. Model 1과 Model 2의 차이.

* Model 1
  * View와 Logic을 JSP 한 페이지에서 처리한다.
  * 간단한 어플리케이션에서는 좋으나 코드가 길어진다.
  * 장점: 구조가 단산하다.
  * 단점: 코드가 어려워져 유지보수하기 어렵다.
* Model 2
  * View와 컨트롤러가 분리된 형태.
  * View에는 처리 로직이 안들어간다.
  * MVC 패턴이라고도 한다.
  * 장점: Model 1에 비해 단순한 구조. 기능이 분리되어 유지보수하기 쉽다
  * 단점: 구조가 복잡하다. Java에 대한 이해가 필요함
